## 🛠 実装における検証記録・トラブルシューティング

実機検証の際、iOS26 Safari特有の表示バグを発見し、その原因究明と対策を行いました。

### 🚨 iOS26 Safariにおけるハンバーガーメニューの表示バグ

#### 【発生条件】
1. ヘッダーが追従している。
2. `<dialog>` タグを使用せず、`div` や `nav` の要素でハンバーガーメニュー（全画面モーダル）を実装している。
3. **iOS26 Safariの下部タブバーが表示されている状態**で、ハンバーガーメニューを展開する。

#### 【現象】
ハンバーガーメニューを展開した際、画面の大半はメニュー画面で覆われるものの、**下部タブバーの背景部分だけ、メニューではなく背面の通常画面（メインコンテンツ）が出てしまう。**

---

### 💡 解決アプローチ

`<dialog>` タグを使った実装へ変更することで解決できます。
その場合、`showModal()` や `close()` を使用するため、JavaScriptの記述もあわせて変更する必要があります。

- `<dialog>` を使った例：https://a-sekiya.github.io/safari-test01/
- `<dialog>` を使わない従来の例：https://a-sekiya.github.io/safari-test01/index02.html
